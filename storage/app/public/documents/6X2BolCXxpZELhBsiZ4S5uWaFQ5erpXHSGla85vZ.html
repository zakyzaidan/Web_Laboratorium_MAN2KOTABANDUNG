Rangkuman Advanced Navigator

Selamat! Anda telah menyelesaikan materi Advanced Navigation. Sejauh ini, Anda telah mengetahui langkah untuk menerapkan Navigator 2.0 pada sebuah aplikasi. Berikut ringkasan materi yang sudah Anda pelajari.

  • Kehadiran Navigator 2.0 tidak menggantikan konsep lama, tetapi melengkapi kekurangan yang ada pada Navigator 1.0.
  • Penerapan Navigator 2.0 ini dapat dilakukan apabila Anda ingin menerapkan fitur navigasi yang memiliki kompleksitas tinggi. Jika tidak, gunakanlah konsep navigasi lama.
  • Berikut alasan Flutter mengusung Navigator 2.0.
    • Menerapkan konsep deklaratif layaknya penggunaan widget Flutter.
    • Memiliki wewenang penuh dalam mengatur navigation stack.
    • Mendukung Flutter Web yang dapat bernavigasi melalui URL atau melihat riwayat halaman yang sudah terbuka sebelumnya.
    • Mampu menerapkan nested navigation di mana satu halaman utama memiliki beberapa halaman di dalamnya, seperti TabBarView, BottomNavigationBar, atau kasus input form dengan banyak step.
    • Menawarkan separation of concerns yang dapat memisahkan logika navigasi dalam sebuah kelas baru di berkas yang berbeda.
  • Navigator 2.0 dibangun atas dua API, yaitu 
    • Page API: Declarative API yang dibangun oleh widget Navigator; dan 
    • Route API: dibangun oleh Widget Router untuk menangani perilaku dari suatu sistem operasi.
  • Widget Navigator memiliki properti yang penting untuk dideklarasikan, yaitu
    • key: identifier atau tanda pengenal untuk mengatur segala urusan navigasi yang ada di naungan widget Navigator. 
    • pages: kumpulan Page yang tersimpan pada navigation stack. Semua widget atau halaman yang Anda punya disusun pada properti ini.
    • onPopPage: sebuah callback yang akan berjalan ketika method pop dipanggil.
  • Callback onPopPage akan berjalan ketika terjadi suatu aksi tertentu, antara lain
    • ketika tombol Back Button aplikasi ditekan;
    • ketika tombol System Back Button ditekan. Biasanya terjadi pada perangkat Android dan Web; dan
    • pemanggilan method pop pada navigasi dengan konsep imperatif.
  • Berikut beberapa kasus yang tidak bisa diatasi oleh Page API.
    • Menekan System Back Button pada Android.
    • Memperbarui URL pada platform Web.
    • Mengatur rute awal pada skenario deep linking, biasanya terjadi pada perangkat Android dan iOS.
    • Menambahkan rute baru pada skenario deep linking, biasanya terjadi pada perangkat Android dan iOS.
  • Berikut alur kerja Router API dalam mengelola rute halaman.
    Perhatikan keterangan dari alur kerja di atas.
    • Pengguna menekan tombol pada halaman teratas pada navigation stack.
    • Aksi tap tombol akan menjalankan perubahan state aplikasi.
    • Widget Router mendengarkan perubahan state aplikasi.
    • Widget Router membangun ulang navigation stack yang ada di widget Navigator.
  • Berikut komponen penyusun Router API.
    • RouteInformationProvider: kelas abstract yang menyediakan informasi halaman untuk widget Router.
    • RouteInformationParser: kelas abstract untuk memilah informasi halaman.
    • RouterDelegate: kelas abstract yang digunakan widget Router untuk membangun ulang navigation stack.
    • BackButtonDispatcher: kelas untuk menangani perilaku System Back Button pada sistem operasi.
  • User experience atau pengalaman pengguna dalam menggunakan aplikasi Web berbeda dengan platform lain. Aplikasi Web menyediakan address bar dan tombol back dan forward untuk bernavigasi ke halaman tertentu.
  • Berikut adalah contoh URL umum pada suatu aplikasi Web.
    Perhatikan penjelasan dari komponen sebuah URL.
    • Scheme: jenis protokol komunikasi sebuah host.
    • Host: nama domain atau penyedia layanan web.
    • Port: nomor protokol sebuah web.
    • Path: lokasi atau direktori suatu web server.
    • Query: tempat untuk melewatkan data berupa string.
  • Ada hal yang harus diperhatikan dalam membuat aplikasi dengan konsep navigasi secara deklaratif, antara lain
    • mendefinisikan konfigurasi setiap halaman yang ada di dalam aplikasi (PageConfiguration);
    • membuat state aplikasi untuk memperbarui navigation stack (RouterDelegate);
    • menangani method pop() apabila kembali ke halaman sebelumnya (onPopPage);
    • menentukan halaman yang akan ditampilkan ke layar berdasarkan perubahan state (Navigator);
    • menerapkan method untuk memilah URL dan mengembalikan konfigurasi halaman (parseRouteInformation);
    • memperbarui state aplikasi berdasarkan konfigurasi halaman (setNewRoutePath);
    • membuat method lain yang mengembalikan konfigurasi halaman berdasarkan state saat ini (currentConfiguration); dan
    • mengembalikan struktur URL ke address bar berdasarkan konfigurasi halaman (restoreRouteInformation).
  • Ada beberapa library yang dapat digunakan untuk menerapkan declarative navigation, seperti go_router, beamer, dan routemaster. Library tersebut merupakan library yang berlabel Flutter Favorite yang direkomendasikan untuk menerapkan declarative navigation.
